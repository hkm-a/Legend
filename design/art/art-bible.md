## 1. Visual Identity Statement

**Visual Rule**

所有重要成长反馈都必须像“火光照亮铁器”一样：热、亮、直接、可读；玩家第一眼就要知道哪里有价值、为何值得捡、捡到后变强了什么。

### Supporting Visual Principles

#### 1. 掉落先于世界

地图、怪物、特效和环境氛围都必须服务于“掉落可见性”。装备、金币、稀有物、强化材料等成长奖励在画面中拥有最高视觉优先级：更高对比度、更明确轮廓、更集中的赤金/火光色、更清晰的动效节奏。环境可以厚重、昏暗、怀旧，但不能抢走掉落物的注意力。

**服务支柱**：爆装有戏、稳刷不断流  
**Design Test**：当“地图氛围表现”和“掉落识别度”冲突时，选择掉落识别度；当环境细节、地面纹理、怪物尸体或特效让掉落物不够突出时，削弱环境，强化掉落。

#### 2. 成长必须有热量

所有与角色变强相关的反馈都应带有“热、亮、锻造、点燃”的视觉语言：赤金高光、火星、熔铁般的边缘光、短促但有重量的闪烁、装备落地时的金属感与冲击感。成长不是冷冰冰的数值变化，而是像武器被重新烧红、淬火、成形一样，让玩家感到力量正在被锻造出来。

**服务支柱**：每次登录都带走成长、爆装有戏  
**Design Test**：当成长反馈只是普通 UI 数字变化时，增加热量感、光感或锻造感；当某个奖励无法让玩家在 1 秒内感到“这东西重要”时，提高其赤金高光、动效强度或声音/视觉联动等级。

#### 3. 现代清晰度包裹经典骨架

视觉表现必须保留传奇式的粗粝、暗色地图、职业辨识、装备崇拜和地图推进感，但所有信息呈现应使用现代标准：清晰层级、稳定布局、可读字体、明确状态、低学习成本。现代 UI 是“打磨后的外壳”，不是替换核心体验的风格噱头；不能把传奇的刷怪、爆装、穿戴和地图推进节奏做得过度轻飘、过度二次元化或过度抽象。

**服务支柱**：传奇骨架，现代皮肤、稳刷不断流  
**Design Test**：当“现代化表现”和“传奇识别感”冲突时，保留传奇骨架，用现代清晰度重绘它；当一个界面很漂亮但让玩家更慢判断装备价值、地图目标或战力变化时，降低装饰性，提升信息层级。

### Rationale

“赤金玛法”的视觉身份将未来所有美术决策限制在一个明确方向内：重要成长反馈必须温热、明亮、直接，不能被环境装饰、复杂 UI 或过度风格化表现稀释。它要求暗色地图成为奖励光的舞台，要求装备和掉落永远比背景更重要，也要求现代化只用于提升可读性和爽感，而不是改变传奇刷怪爆装的核心气质。未来评审任何角色、地图、UI、特效或掉落表现时，都应先问：它是否让玩家更快看见价值、更强烈感到成长、更顺畅地继续刷下去？如果答案是否定的，即使画面精美，也不符合本项目的 Visual Identity。

## 2. Mood & Atmosphere

本节定义《新玛法：觉醒客户端》在主要游戏状态下的情绪目标、光照气质、氛围关键词、能量等级与关键视觉承载元素。整体原则延续第 1 节 Visual Identity Statement：**暗色世界是底，赤金成长是火；所有重要反馈都应像火光照亮铁器一样热、亮、直接、可读。**

### 2.1 主菜单 / 登录前氛围

**Primary emotion / mood target**  
沉睡中的旧玛法正在重新点燃。玩家应感到这是一个熟悉、粗粝、古老的世界，但新的成长火种已经出现，等待进入。

**Lighting character**  
- 冷暗为底，暖光为焦点。
- 整体低亮度、中高对比。
- 时间感接近黄昏后或深夜前：世界已暗，火光成为唯一可靠指引。
- 画面不应明亮热闹，而应有“旧城门前炉火未熄”的召唤感。

**Atmospheric descriptors**  
古旧、低沉、余烬感、粗粝、被唤醒。

**Energy level**  
低到中。静态压迫感为主，局部火光、旗帜、灰烬或金属反光提供轻微动势。

**Visual atmosphere carrier**  
旧石城门、残破玛法纹章、锈蚀兵器与一处稳定燃烧的锻炉火光。Logo 或进入按钮可由赤金光边轻微照亮，暗示“进入即开始成长”。背景可有缓慢飘落火星或烟尘，但不得喧宾夺主。

### 2.2 城镇 / 安全区

**Primary emotion / mood target**  
安全、整备、交易、修补与下一轮出发前的积蓄。城镇不是舒适明亮的天堂，而是黑暗地图之间的火光据点。

**Lighting character**  
- 暖光占主导，但亮度受控。
- 火把、铁匠炉、摊位灯、门窗光构成局部光池。
- 对比中等：玩家能清楚读取 NPC、功能建筑与道路，但仍保留传奇式暗色底色。
- 时间感偏夜间据点或阴天旧城，火光承担主要视觉组织功能。

**Atmospheric descriptors**  
安全、繁忙、烟火气、金属感、准备出征。

**Energy level**  
中。NPC、玩家、摊位和火光让画面有生活节奏，但不应像节庆场景一样高能。

**Visual atmosphere carrier**  
铁匠铺应是城镇视觉核心之一：炉火、砧台、挂起的武器、被火光照亮的金属边缘。安全区边界可用温暖灯火或铺地纹理暗示“这里可喘息”。仓库、商人、强化相关入口应有比普通建筑更清晰的暖色识别点。

### 2.3 普通刷怪地图

**Primary emotion / mood target**  
稳定刷怪、持续推进、略有危险但可掌控。玩家应进入“不断流”的专注状态：看怪、打怪、等爆、继续走。

**Lighting character**  
- 冷暗环境为主，暖色奖励反馈作为强视觉对比。
- 整体低饱和、低到中亮度。
- 地图本身不应抢夺掉落光效的注意力。
- 时间感多为夜间、地下、阴天、矿洞、荒野深处或旧遗迹内部。

**Atmospheric descriptors**  
阴冷、粗粝、压抑、可控危险、反复推进。

**Energy level**  
中低。普通刷怪地图应支持长时间观看，避免持续高闪烁、高饱和、高噪声。

**Visual atmosphere carrier**  
暗石地面、矿洞岩壁、旧城废墟、低饱和植被、残破路标。小范围环境光源如火把、矿灯、裂缝冷光可以提供空间层次，但不得与掉落赤金光竞争。怪物剪影需在暗底中清楚可读，避免与地形融成一团。

### 2.4 战斗中

**Primary emotion / mood target**  
直接、沉重、清晰、有打击反馈。战斗应让玩家感到自己正在高效率清怪，而不是被复杂视觉噪声包围。

**Lighting character**  
- 环境维持冷暗底色。
- 技能、命中、受击、暴击等反馈使用短促暖光或职业色点亮。
- 对比高于普通行走状态，但持续时间短，避免遮挡怪物、角色和掉落。
- 时间感不改变，战斗光效只是在原本阴暗空间中瞬间点火。

**Atmospheric descriptors**  
直接、沉重、火花四溅、清楚、有压迫。

**Energy level**  
中到中高。普通战斗应有节奏峰值，但不能长期维持 Boss 级视觉强度。

**Visual atmosphere carrier**  
武器命中时的金属火花、短促热光、地面轻微冲击尘。战士偏赤金与铁火，法师偏高亮魔法核心但外缘需受控，道士偏符纸、灵光与药性暖光。命中特效必须遵循视觉层级：角色动作和目标轮廓优先，特效其次，环境最后。

### 2.5 怪物死亡 / 掉落瞬间

**Primary emotion / mood target**  
爆装有戏。玩家必须在第一眼感到“有东西出了”，并能迅速判断价值等级。死亡与掉落是刷怪循环的情绪峰值。

**Lighting character**  
- 掉落瞬间使用最强暖色对比：赤金、热橙、金属白高光。
- 怪物死亡可短暂降低周边环境注意力，使掉落光成为焦点。
- 高对比、高可读、短爆发。
- 时间感像黑暗矿洞里突然露出金矿火光：一瞬间把玩家的注意力拉到地面。

**Atmospheric descriptors**  
爆发、惊喜、热亮、值钱、直接。

**Energy level**  
高，但持续时间短。峰值应集中在死亡帧、掉落生成、稀有物品落地三个节点。

**Visual atmosphere carrier**  
装备落地的赤金光柱、金属弹跳高光、短促火星扩散。普通掉落可用低强度暖光；高价值掉落必须有更强的轮廓、光柱、粒子密度或音画同步感。掉落位置应优先清晰，避免被尸体、地形、技能残留遮挡。

### 2.6 拾取 / 装备提升

**Primary emotion / mood target**  
变强已经发生。玩家不只是“拿到了东西”，而是立刻感到自己的成长被点燃、被锻造、被确认。

**Lighting character**  
- 拾取使用短促暖光回流：从地面奖励到角色 / UI 的视觉路径。
- 装备提升使用更稳定、更有仪式感的赤金高光。
- 对比中高，重点在“结果可读”而非纯炫耀。
- 时间感像铁器从炉火中取出：短暂炽亮，随后回到可操作状态。

**Atmospheric descriptors**  
成长、点燃、扎实、奖励感、被确认。

**Energy level**  
中到高。普通拾取中等，装备替换、战力提升、稀有装备入包应达到高能反馈。

**Visual atmosphere carrier**  
拾取轨迹光：赤金粒子或细线从掉落点流向角色 / 背包提示。装备提升提示可使用“铁器淬火后的亮边”语言：装备图标边缘被火光扫过，数值增长处有暖色脉冲。若有战力或属性提升，应让提升数字先于装饰元素可读。

### 2.7 精英怪 / Boss 场景

**Primary emotion / mood target**  
危险、压迫、值得一打。玩家应感到这里的风险更高，但也更可能爆出真正有价值的东西。

**Lighting character**  
- 环境更暗、更集中、更有方向性。
- Boss 或精英怪拥有明确轮廓光，通常可用冷暗环境加局部危险色。
- 奖励预期可通过场景中的赤金矿脉、祭坛火光、宝箱残骸或装备残片暗示。
- 对比高，空间中心应明确，不让玩家迷失目标。

**Atmospheric descriptors**  
压迫、危险、稀有、仪式感、值得冒险。

**Energy level**  
中高到高。进入场景时中高，战斗爆发时高，死亡掉落瞬间达到最高。

**Visual atmosphere carrier**  
Boss 场地中心光：祭坛、矿洞深处的熔火裂缝、旧王座、血色旗帜或巨型锻炉残骸。精英怪脚下或背后可有低频脉动光，提示其不同于普通怪。Boss 死亡时掉落光效必须压过场景危险光，形成“风险转化为奖励”的视觉叙事。

### 2.8 失败 / 死亡

**Primary emotion / mood target**  
损失、冷却、被迫中断，但不绝望。玩家应明确知道自己失败了，同时仍被引导回到“修整后继续刷”的循环。

**Lighting character**  
- 暖光撤退，冷暗重新占据画面。
- 低饱和、低亮度，红色只用于危险和死亡确认，不应大量铺满。
- 对比降低，动作和环境进入短暂静止感。
- 时间感像炉火被风压低：不是熄灭世界，而是提醒玩家需要重新点燃。

**Atmospheric descriptors**  
冰冷、中断、沉重、克制、可恢复。

**Energy level**  
低。死亡瞬间可有短促冲击，但随后快速降能，给玩家清楚选择下一步的空间。

**Visual atmosphere carrier**  
角色倒地后的冷色阴影、熄灭的火星、降低饱和的场景。复活、回城、重试按钮可保留受控暖光，作为“重新点燃”的出口。若显示损失或惩罚信息，应使用清楚的视觉层级，避免恐吓式过度红色。

### 2.9 背包 / 装备整理界面

**Primary emotion / mood target**  
盘点收获、比较价值、整理成长。这里应像把战利品放到铁匠台上检视：清楚、扎实、有装备崇拜感。

**Lighting character**  
- UI 底色偏暗、低饱和，物品图标与品质光成为视觉主角。
- 暖光用于装备、强化、可提升项；冷灰用于背景、未选中格、普通信息。
- 对比应服务于可读性：数值、品质、可装备状态、提升方向必须第一时间明确。
- 时间感接近室内铁匠铺或仓库灯光：安静、稳定、便于判断。

**Atmospheric descriptors**  
清楚、扎实、可比较、战利品感、锻造感。

**Energy level**  
中低。整理界面应长时间可用，避免过度动效；只有获得高价值装备、装备替换、强化成功时短暂升能。

**Visual atmosphere carrier**  
暗金属边框、旧皮革 / 石板底纹、格子内的受控品质光。当前推荐装备或可提升装备可有细微赤金扫光，但不能让整个背包同时发亮。装备详情面板可使用“铁匠台检视”构图：装备图标像被放在暖光下，属性提升以清楚的热色数字呈现。

## 3. Shape Language

### 3.1 Overall Grammar：经典骨架，热光切面

《新玛法：觉醒客户端》的形状语言以“粗粝、稳定、可读”为基础：大形保持经典传奇式的厚重轮廓，小形用现代化清晰切面提升识别。整体避免轻飘、圆润、过度装饰的幻想感，优先使用厚矩形、斜切面、钝角、破损边、铁器般的硬边结构。

核心规则：**世界用冷硬块面承载，成长反馈用赤金锐光刺穿。**  
因此，场景与角色底形偏沉、暗、宽；掉落、升级、装备强化等成长反馈使用更亮、更尖、更集中的形状，像火星、熔线、刀刃高光一样立即抓住视线。

### 3.2 Character Silhouettes

**玩家角色**：轮廓应稳定、直立、可辨职业。战士偏宽肩、重心低、武器外扩；法师偏长袍纵向、法杖形成高识别竖线；道士偏中等体量，以符纸、铃、短杖或召唤物形成辅助轮廓。玩家必须在怪群中保持清晰外形，武器和肩部是第一识别点。

**普通怪物**：轮廓压低、粗短、群体化。使用驼背、爪、角、残破肢体等不规则边缘，但不得比玩家更复杂。普通怪的形状职责是制造刷怪密度，不抢掉落和玩家动作的视觉优先级。

**精英 / Boss**：使用放大的三角威胁形与外扩结构：角、肩甲、巨爪、背刺、披风残片。Boss 轮廓必须在缩放视角下仍能一眼区别于普通怪；其危险部位应有明确方向性，例如前冲尖角、横扫巨臂、中心发光核心。

**NPC**：城镇 NPC 以稳固、垂直、温暖轮廓为主。铁匠、商人、仓库等功能 NPC 可加入工具外形：铁砧、背包、货架、灯笼。NPC 不应有强攻击性外扩形，避免与怪物或玩家技能混淆。

### 3.3 Environment Geometry

城镇是“火光据点”：几何偏稳定、厚墙、木梁、石台、炉口、拱门，形成安全的围合感。可使用方形与拱形组合，火盆、灯笼、熔炉作为温暖视觉锚点。

刷怪地图是“阴冷粗粝”：地形使用破碎石块、斜裂纹、低矮残墙、枯木、洞穴边缘。形状应横向延展，服务连续移动和刷怪节奏；避免过多竖向遮挡影响战斗读图。

掉落区域周围的地面形状应保持低噪声，让装备、金币、材料的亮色轮廓能从暗底中跳出。

### 3.4 UI Shape Grammar

UI 采用“铁匠台盘点战利品”的形状语法：面板厚重、边框硬朗、格子清晰。背包、装备栏、掉落提示使用矩形网格与金属包边，允许轻微斜切角强化铁器感。

主要按钮使用厚矩形或斜切矩形，强调直接可点击；次级按钮更薄、更暗、更少高光。成长反馈 UI，如升级、爆装、强化成功，应使用赤金外发光、锐角光束、火星粒形，形成高层级视觉峰值。

### 3.5 Hero vs Supporting Shapes

**Hero Shapes**：装备掉落、角色升级、Boss、玩家武器、强化结果。它们可以更亮、更尖、更集中，拥有最高对比度和最明确轮廓。

**Supporting Shapes**：地面、墙体、普通怪、常驻 UI 边框、装饰物。它们应更钝、更暗、更低饱和，避免抢夺爆装瞬间的注意力。

设计原则：**所有非成长元素都在为成长火光让路。**

### 3.6 Emotional Meaning

厚矩形与钝角代表可靠、经典、可持续刷怪；破碎边与冷硬斜面代表玛法世界的危险和旧时代质感；赤金尖光、火星、熔线代表成长、爆装、变强。圆形仅少量用于光源、金币、宝石和温暖据点，不作为主要世界语法，避免削弱沉重感。

### 3.7 Design Tests

1. 黑白剪影下，玩家、普通怪、Boss、NPC 是否能在 1 秒内区分。
2. 去掉颜色后，掉落物是否仍比地面和普通怪更醒目。
3. 战斗截图缩小到 50% 时，玩家武器方向和怪物威胁方向是否清楚。
4. 背包界面是否像整理战利品，而不是轻薄网页 UI。
5. 每个成长反馈是否具备“热、亮、直接、可读”的火光特征。

## 4. Color System

### 4.1 Primary Palette

本项目色彩系统以“暗色世界 + 赤金成长”为核心。世界负责压低情绪、提供刷怪背景；赤金负责爆装、升级、强化等成长反馈，必须第一时间从画面中跳出。

| 色名 | 近似 Hex | 用途 / 含义 |
|---|---:|---|
| 深铁黑 | `#151515` | 世界底色、UI 底板、背包背景；提供低干扰承载面 |
| 冷石灰 | `#3A3D40` | 地面、墙体、普通怪物阴影、非重点信息 |
| 旧皮革棕 | `#4A3424` | 背包、商店、角色面板的次级材质 |
| 暗金属边 | `#6A563A` | UI 边框、格子线、按钮描边，保留经典厚重感 |
| 赤金火光 | `#F0A020` | 成长主色：掉落、升级、强化成功、关键奖励 |
| 灼热橙红 | `#D94A1E` | 战斗命中、危险警示、Boss 压迫、爆发瞬间 |
| 冷蓝阴影 | `#2B3E52` | 夜色、矿洞、刷怪地图冷调、未激活状态 |

赤金火光是唯一的“成长主权色”。除非表达成长、奖励或关键完成，不应大面积使用赤金，避免稀释爆装瞬间的戏剧性。

### 4.2 Semantic Color Usage

- **危险**：灼热橙红 `#D94A1E`，用于受击、Boss 技能预警、血量危急。必须配合尖角形状、闪烁或震动。
- **奖励**：赤金火光 `#F0A020`，用于掉落光柱、结算奖励、稀有道具获得。
- **稀有度**：使用独立品质色，但高品质可叠加赤金描边或光晕。
- **可交互**：暗金属边 + 微弱暖光；悬停 / 选中时提升亮度，不使用纯蓝科技感。
- **安全**：城镇暖黄 `#C98A45` 与低对比棕色，表现据点、修整、交易。
- **普通信息**：冷石灰或米灰 `#B8B0A0`，保持可读但不抢奖励焦点。
- **成长完成**：赤金 + 短促亮化 + 锐利外发光，必须像“火光照亮铁器”。

### 4.3 装备品质颜色规则

品质色用于图标边框、物品名称、掉落光效和背包格子强调；不可只依赖颜色，需配合边框层级、星标 / 角标、光效强度。

| 品质 | 颜色 | 规则 |
|---|---:|---|
| 普通 | `#B8B0A0` 米灰 | 无光效，清晰可读 |
| 优秀 | `#4FB06D` 绿 | 轻微描边，可有小型角标 |
| 稀有 | `#3D8CFF` 蓝 | 明显边框，掉落有短光闪 |
| 史诗 | `#A85CFF` 紫 | 外发光、图标高亮、名称加粗 |
| 传说 | `#F0A020` 赤金 | 掉落光柱、火花、强音效、屏幕级反馈 |
| 神话 / 特殊 | `#FF3D2E` 赤红金 | 仅用于极少数目标，需配专属轮廓和动效 |

传说以上品质不得仅通过更亮的黄色区分，必须加入形状、动画和音效层级，保证爆装有戏。

### 4.4 Per-Area Color Temperature Rules

- **城镇**：暖色据点。主温度偏暖，使用旧皮革棕、暖黄灯光、暗金属边。玩家应感到“整理战利品、准备再刷”。
- **矿洞 / 刷怪地图**：冷暗低饱和。以深铁黑、冷石灰、冷蓝阴影为主，怪物和地形压低对比，让掉落与命中特效成为视觉焦点。
- **Boss 区域**：冷暗底 + 局部橙红威胁。Boss 本体可带低饱和红 / 铜色，技能预警必须比环境更热、更尖锐。
- **UI 界面**：暗金属与旧皮革为底，赤金只用于成长结果、确认收益、重要按钮或当前目标，不可铺满界面。

### 4.5 UI Palette Divergence

世界色更冷、更脏、更低饱和；UI 色更清晰、更规整、更高可读。世界负责“玛法仍然危险”，UI 负责“玩家知道自己得到了什么”。

UI 可以使用更稳定的暗金属、皮革和米灰文字，但要避免现代纯白、荧光蓝、大面积渐变玻璃感。按钮、格子、面板应像铁匠台上的清点区域：厚重、清楚、可快速扫视。

### 4.6 Colorblind Safety

以下语义不得只靠颜色表达，必须有形状、图标、动效或音效备份：

- 装备品质：边框层级、角标、光效强度、掉落音效。
- 危险预警：尖角 / 扇形区域、闪烁频率、屏幕震动。
- 可交互对象：描边、手型 / 按钮状态、悬停反馈。
- 成长完成：数字跳动、火花、音效、短暂停顿。
- 安全 / 危险区域：地图图标、入口标记、环境音差异。

### 4.7 Practical Design Tests

1. **灰度测试**：去掉颜色后，掉落、Boss 危险、按钮状态仍应可识别。
2. **三秒识别测试**：玩家在三秒内能看出“哪里掉了好东西”。
3. **赤金占比测试**：普通刷怪画面中赤金面积应很小；奖励瞬间才爆发。
4. **背包扫视测试**：玩家能快速区分品质、可装备、可出售、已强化。
5. **冷暖对比测试**：地图越危险越冷暗，成长反馈越热亮。
6. **经典感测试**：即使使用现代清晰 UI，也不能出现科幻霓虹、玻璃拟态或手游过度彩虹化。

## 5. Character Design Direction

本节定义《新玛法：觉醒客户端》的玩家角色、敌人、Boss、NPC、动作与装备成长视觉方向。所有角色设计必须服务第 1 节的核心规则：**成长反馈像火光照亮铁器一样热、亮、直接、可读**。角色本体要有经典传奇识别度，怪物要支撑刷怪密度，装备与掉落要成为画面中最诱人的成长焦点。

### 5.1 Player Character Class Direction

长期职业方向保留经典传奇骨架：战士、法师、道士。三职业必须在黑白剪影、小尺寸画面和混战状态下能快速区分，职业差异优先体现在轮廓、装备材质、动作节奏和能量来源，而不是依赖 UI 名称或颜色标签。

**战士**  
战士是厚重、近身、铁器感最强的职业。轮廓以宽肩、低重心、重武器外扩为核心，材质偏旧铁、暗钢、皮革、赤金火光边缘。战士动作应短促、有重量、有明确命中点，视觉关键词是“铁、火、冲击、压迫”。

**法师**  
法师以长袍纵向轮廓、法杖或魔法核心形成识别。色彩可使用冷蓝、紫、白热能量，但亮度必须受控，不能长期压过掉落赤金光。法师视觉关键词是“纵向、聚能、爆发、脆弱但高亮”。

**道士**  
道士以符纸、铃、药囊、骨饰、短杖或召唤物形成辅助轮廓。装备应更实用、更旧、更混合，避免过度华丽。能量表现可偏符纸暖光、灵光、药性绿色或低饱和神秘色，但仍需与成长赤金区分。道士视觉关键词是“符、药、灵、旧装备、辅助轮廓”。

**Phase 1 character scope**  
Phase 1 的 30 秒刷怪爆装技术切片建议只制作单一玩家角色，并采用“战士基底”。原因是战士最适合验证近战刷怪、怪物死亡、装备掉落、拾取和穿戴反馈：武器方向清楚，命中反馈直接，装备成长最容易通过材质、轮廓和火光表现。

### 5.2 Enemy Design Direction

普通怪物应服务稳定刷怪节奏，不能抢走玩家、掉落和装备反馈的视觉层级。它们可以粗粝、阴冷、危险，但必须保持低饱和、中低亮度和可批量识别。

**普通怪**  
普通怪使用压低、粗短、不规则的剪影，如驼背、爪、角、残破肢体、破布或硬壳。颜色偏冷暗、脏灰、低饱和棕绿或石灰。普通怪的细节密度必须低于玩家角色和掉落物；死亡后尸体或残留不能遮挡掉落。

**精英怪**  
精英怪在普通怪基础上增加一个明确识别符号，例如赤暗裂纹、背刺、发光眼、额外肩甲、脚下低频脉动或更大的武器。精英识别应在 1 秒内成立，但其光效不能比高价值掉落更诱人。

**Boss**  
Boss 使用大剪影、厚块面、慢动作和压迫感。轮廓可包含巨角、巨爪、背刺、披风残片、核心发光点或大型武器。Boss 的危险光可使用橙红、暗红或铜色，但死亡掉落瞬间必须由掉落赤金光夺回视觉主权，形成“风险转化为奖励”的画面结果。

### 5.3 NPC Design Direction

NPC 负责城镇功能识别和安全区氛围，应稳定、垂直、温暖、可亲近，但不能过度华丽或像战斗角色。

**铁匠**  
铁匠是城镇视觉核心。必须具备炉火、铁砧、厚围裙、工具、被火光照亮的金属边缘。铁匠与装备成长强绑定，可使用最明显的暖光，但仍应低于爆装和强化成功的峰值亮度。

**商人**  
商人使用旧皮革、布包、木箱、挂货、账本或小灯作为识别元素。视觉重点是交易和补给，不应带强攻击轮廓。

**仓库 / 功能 NPC**  
仓库与功能 NPC 使用箱柜、账册、钥匙串、柜台、标牌等道具强化识别。轮廓应稳固、规整，方便玩家在城镇中快速定位功能入口。

### 5.4 Animation & Pose Language

角色动作风格应短促、清楚、有重量，不追求过度表演。所有动作必须优先服务读图和刷怪节奏：玩家应能快速判断攻击方向、命中时机、怪物受击状态和掉落位置。

- **待机**：低幅度、低噪声，保留呼吸、武器重量或布料轻动。
- **移动**：节奏直接，脚步和重心清楚，避免滑步感。
- **攻击**：起手短、命中帧明确、收招干净，武器轨迹可用短促火星或残影强调方向。
- **受击**：反馈清楚但不长时间打断刷怪节奏。
- **死亡**：普通怪死亡应快速降层级，为掉落让出视觉焦点；Boss 死亡可更有仪式感，但峰值应转移到掉落。
- **拾取 / 装备提升**：角色和 UI 可有短促暖光响应，让玩家感到成长已经进入角色身上。

### 5.5 Gear Progression Visual Rules

装备成长必须能被玩家看见。成长变化不只来自数值，也来自轮廓、材质、局部发光、动作残影和 UI 图标一致性。

装备视觉成长建议从“旧铁”推进到“赤金火光”：

1. **初始 / 普通装备**：旧铁、破皮革、暗灰布料，轮廓简单，无明显光效。
2. **优秀 / 稀有装备**：更完整的金属边、少量色彩识别、武器轮廓更清楚。
3. **史诗装备**：增加局部发光、装饰切面、材质对比和更强图标边框。
4. **传说装备**：赤金高光、火星、明确外轮廓变化、命中或移动时有受控残影。
5. **特殊 / 神话装备**：需要专属剪影或专属动效，但必须极少使用，避免稀释长期追求感。

装备在角色身上、背包图标、掉落物表现和详情面板中应共享同一视觉语言。玩家看到掉落、捡起、打开背包、穿上装备时，应感觉是同一件战利品在不同界面中延续，而不是四套互不相干的图像。

### 5.6 Character Design Tests

1. **黑影测试**：玩家、普通怪、精英怪、Boss、铁匠、商人能否在黑色剪影下区分。
2. **灰阶测试**：去掉颜色后，装备成长层级、敌人威胁层级和 NPC 功能是否仍可判断。
3. **小尺寸测试**：角色缩小到实际游戏镜头尺寸后，职业、武器方向、怪物类别是否仍清楚。
4. **亮度竞争测试**：同屏出现玩家、怪物、Boss 危险光和掉落物时，掉落是否仍拥有最高成长焦点。
5. **同屏压力测试**：多个普通怪围攻时，画面是否仍能读出玩家位置、攻击方向和掉落点。
6. **经典识别测试**：不看 UI 名称时，玩家是否仍能感到这是传奇骨架下的战士 / 法师 / 道士，而不是完全陌生的幻想职业。

### 5.7 Phase 1 Sprite Production Clarifications

Phase 1 角色生产只制作“战士基底”的基础外观，不制作完整装备外观替换系统。装备成长在 Phase 1 主要通过地面掉落、背包图标、装备详情、穿戴反馈、属性 / 战力提示和短促赤金高光表达；角色身上只允许少量武器边缘高光或基础武器剪影变化，不要求每件装备都改变角色 Sprite。

角色与怪物方向数以 Phase 1 实际移动 / 攻击需求为准：优先 4 方向或必要方向 + 镜像复用，不默认制作 8 方向全套。若后续决定对齐 OpenMir2 的方向规则，应在角色 production sheet 中单独标注方向数、镜像规则、攻击方向和脚底锚点。

所有角色 / 怪物 sprite sheet 必须提供：正面、背面、侧面或等价方向参考；脚底中心 pivot；默认镜头角度；武器挥砍外扩范围；与 64×64 矿洞 tile 的比例预览。战士默认在实际镜头下应约占 1.5–2.5 个 tile 的视觉高度，具体比例以点击移动、怪物围攻和掉落识别测试为准。

## 6. Environment & Level Art

本节定义《新玛法：觉醒客户端》Phase 1 环境与关卡美术方向。Phase 1 首个环境锁定为矿洞刷怪图，服务 30 秒离线刷怪爆装切片。矿洞不是探索型大地图，而是一个让玩家稳定完成点击移动、打怪、掉落、拾取、装备判断和穿戴变强的刷怪容器。

整体规则延续第 1 节视觉身份：**暗色世界为底，赤金成长为火。** 环境必须冷暗、低饱和、粗粝，并主动为掉落、怪物、角色和成长反馈让出视觉优先级。

### 6.1 Core Environment Direction：矿洞刷怪图

Phase 1 矿洞应是一个清楚、可刷、可验证的战斗场地，而不是复杂迷宫或叙事探索空间。玩家进入后应能立刻看见移动方向、怪物来源、战斗空场和掉落位置。

核心规则：

- **不阻断刷怪流**：路线清楚，不做复杂迷宫。
- **衬托爆装**：掉落必须比地面、碎石、尸体和装饰物更先被看见。
- **保留传奇骨架**：低饱和、压迫、旧矿洞刷怪场质感。
- **暗色世界为底，赤金成长为火**：环境压低，成长反馈点亮。

### 6.2 Architecture & Environment Style

矿洞是“旧玛法文明的劳作伤口”：半人工、半坍塌，曾用于采铁和冶炼原料运输，后来被战争、灾难和怪物侵蚀。它应有可识别的人类开采痕迹，但这些痕迹必须服务空间边界、方向感和气氛，而不能干扰拾取与战斗。

**视觉元素**

- 木支架、断梁、铁轨、矿车、绳索、矿镐、矿石堆。
- 塌方、焦痕、爪痕、暗血痕、破碎甲片。
- 少量火盆或余烬，暗示火种未灭，但不能抢赤金掉落光。

**形体规则**

- 使用粗粝厚块面、破碎石块、斜裂纹、低矮残墙、洞穴边缘。
- 高遮挡物靠边，不进入主战斗区。
- 禁止大面积荧光矿物、宝石堆、亮色武器残片，以免混淆掉落。

### 6.3 Texture Philosophy

Phase 1 采用**手绘 / 半手绘风格化材质**，不走完整 PBR 写实。

原因：

- Godot 2D / CanvasItem 管线更适合 sprite / tile / atlas 的可读控制。
- Phase 1 需要掉落和角色读图稳定，不依赖复杂动态光照。
- 照片 PBR 高频噪声会降低掉落可见性。
- 纯平卡通太轻，无法承载玛法的粗粝铁器感。

**材质规则**

- 地面低噪声、低饱和、低高光，是掉落承托层。
- 洞壁更厚重、更破碎，使用冷石灰、深铁黑、暗蓝灰。
- 木材偏旧皮革棕、焦褐、灰棕。
- 金属偏黑铁、氧化棕、暗金属边；环境金属不能比装备掉落更像宝物。
- 火光只做局部引导，小矿洞明显火光点控制在 2–4 个。

### 6.4 Prop Density Rules

Phase 1 矿洞采用低到中密度，保证刷怪与拾取空间优先。

| 区域 / 元素 | 占比目标 | 规则 |
|---|---:|---|
| 可战斗 / 可拾取干净区域 | 约 60% | 主战斗区保留清晰中型空场 |
| 边界与不可通行结构 | 约 25% | 用岩壁、塌方、矿车、木梁组织边界 |
| 叙事与装饰道具 | 约 10% | 只放在不干扰移动和拾取的位置 |
| 火光 / 重点视觉点 | 约 5% | 2–4 个受控暖光点，不抢掉落 |

具体规则：

- 怪物刷新点周围无高遮挡物。
- 常见掉落区域地面必须干净。
- 矿车、木梁、箱子用于边界，不散落在拾取路线上。
- 小碎石放边缘，战斗中心只允许极少量低对比碎石。

后续地图可按主题调整密度：旧城废墟可提高结构密度，但不能做复杂迷宫；城镇安全区可更密、更暖，但 NPC、背包、装备交互仍优先于背景装饰。

### 6.5 Environmental Storytelling

环境叙事原则：**先保证玩法可读，再表达地点身份，最后才放历史细节。**

Phase 1 矿洞讲述五件事：

1. **曾经被人类开采**：铁轨、矿车、木支架、矿镐、矿石筐。
2. **曾发生灾难或战斗**：塌方、断梁、焦痕、散落物、暗血痕。
3. **怪物占据这里**：爪痕、骨骸、破布巢穴、腐蚀痕迹。
4. **与装备成长有关**：黑铁矿、废弃炉具、断剑残片，但不能像可拾取装备。
5. **仍有火种**：少量火盆或余烬呼应赤金成长主题。

禁止：

- 大段碑文或说明牌。
- 过多尸体和血迹。
- 大面积发光符文。
- 会被误认为金币、药水、装备的亮色装饰。
- 与 Phase 1 战斗无关的复杂机关暗示。

### 6.6 Level Readability / Gameplay Constraints

**掉落可见性**

- 掉落区域地面低噪声、低饱和。
- 掉落周围不能有像装备、金币、药水的小道具。
- 稀有掉落赤金反馈必须压过火盆和金属高光。
- 怪物死亡位置周围保留视觉净空。

**点击移动**

- 可走地面与不可走边界必须清楚。
- 装饰物不可造成频繁卡边、绕路或误点。
- 不做主战斗区细碎障碍。

**阻挡 / Y-sort**

- 视觉阻挡与实际碰撞一致。
- 高遮挡物放边缘。
- 主战斗区只放低矮道具。
- 掉落不能被墙、矿车、木梁遮住。

**刷怪路线**

- 一个主空场。
- 2–3 个怪物刷新 / 进入方向。
- 一条清楚主移动轴线。
- 30 秒内不要求跨越复杂房间。

### 6.7 Phase 1 Minimum Environment Scope

必须包含：

- 1 个主战斗空场。
- 1 个玩家出生 / 进入点。
- 2–3 个怪物刷新或进入方向。
- 明确不可通行边界。
- 低噪声矿洞地面、洞壁 / 岩壁边界、塌方或碎石边界。
- 木支架、铁轨、矿车、矿石堆、木箱、工具、火盆、少量怪物侵蚀痕迹。
- 2–4 个局部暖光点。
- 能验证 30 秒内看见怪物、击杀反馈、掉落、拾取反馈。

不包含：

- 多层矿洞。
- 复杂机关。
- 可破坏环境。
- 动态坍塌。
- 大型 Boss 场景。
- 多主题区域。
- 大量 NPC。
- 高复杂度前景遮挡。
- 完整 PBR 材质流程。
- 大范围探索路线。

### 6.8 Practical Review Tests

1. **掉落优先测试**：50% 缩图仍能第一眼看到掉落，火盆和矿点不能抢稀有掉落。
2. **点击移动测试**：连续 30 秒点击移动、打怪、拾取，不应误点、卡边、绕路。
3. **遮挡测试**：角色、怪物、掉落放在矿车、木支架、岩壁附近时不被吞掉。
4. **低噪声地面测试**：地面装饰不能像掉落。
5. **经典识别测试**：不看 UI，也能看出这是传奇式玛法矿洞，而不是通用奇幻洞穴.

### 6.9 TileMap / Y-sort Production Clarifications

矿洞环境资产必须按 Godot TileMap、Y-sort 和点击移动需求拆分，不允许只交付“看起来像矿洞”的整张插画。

最低 tile 分类：

| 分类 | 用途 | 视觉规则 |
|---|---|---|
| 可走地面 | 主移动、战斗、掉落区域 | 低噪声、低饱和、边界平整，不像掉落物 |
| 不可走边界 | 岩壁、塌方、墙体、矿车阻挡 | 明度和形状必须清楚区别于可走地面 |
| Y-sort 实体 | 木支架、柱、矿车、墙角等可遮挡物 | 必须标注脚底 / 接地点和遮挡范围 |
| 前景遮挡 | 洞口前沿、高木梁、屋檐式遮挡 | 只放边缘，不覆盖常见掉落区域 |
| 装饰地表 | 裂纹、小碎石、矿渣 | 低对比，不得像金币、装备、药水或材料 |
| 环境火光 | 火盆、余烬、矿灯 | 亮度低于稀有掉落赤金光 |

每套矿洞环境交付时必须附一张 64×64 tile 比例预览图，显示：玩家、普通怪、掉落物、可走地面、不可走边界、Y-sort 遮挡物和火光装饰同屏关系。若该预览中掉落物不明显，环境资产必须降噪或降亮度。

## 7. UI/HUD Visual Direction

本节定义《新玛法：觉醒客户端》的 UI/HUD 视觉方向。UI 必须服务 Phase 1 的 30 秒闭环：**点击移动 → 打怪 → 掉落 → 拾取 → 背包判断 → 穿戴 → 看到变强**。

UI 不是装饰层，而是玩家判断战利品、确认成长、继续刷怪的工具。整体应像铁匠台上盘点战利品：暗金属、旧皮革、石板、米灰文字，赤金只用于掉落、成长、穿戴成功和关键确认。

### 7.1 Core UI Rule：战利品判断工具优先

UI/HUD 的第一职责是让玩家更快完成刷怪爆装循环，而不是展示复杂装饰。所有面板、按钮、Tooltip、掉落提示和装备对比都必须优先回答以下问题：

1. 我现在是否安全？
2. 地上出了什么？
3. 哪个掉落最重要？
4. 背包里什么是新的？
5. 这件装备能不能穿？
6. 穿上后强在哪里？
7. 我是否已经变强？

如果厚重装饰、材质噪声、边框细节、粒子动效或背景纹理影响这些判断，应删减装饰而不是牺牲可读性。

### 7.2 HUD Composition

Phase 1 HUD 采用 screen-space 为主，少量 diegetic 反馈强化掉落和拾取结果。常驻 HUD 保持克制，只显示刷怪闭环需要的信息。

**常驻 HUD**

- 玩家血量 / 生存状态。
- 当前目标名称与血量。
- 背包入口与主要快捷键提示。
- 必要目标提示，例如“击杀怪物”“拾取掉落”“打开背包穿戴”。

**事件型 HUD**

- 掉落名称、品质、可拾取提示。
- 拾取成功提示。
- 新装备入包提示。
- 装备可提升提示。
- 穿戴成功与属性提升提示。
- 错误提示，例如背包已满、无法穿戴、距离太远。

事件型 HUD 只在事件发生时出现，结束后快速退场，避免长期占用战斗视野。赤金不能泛用到普通按钮和边框，否则会削弱“爆装有戏”。

### 7.3 Combat HUD Visual Priority

战斗 HUD 必须让掉落和成长反馈压过普通战斗噪声，但不能遮挡角色、怪物轮廓和物品名称。

视觉优先级从高到低：

1. 稀有掉落 / 可提升装备。
2. 低血量等生存风险。
3. 当前目标血量 / 选中反馈。
4. 普通掉落。
5. 普通伤害数字。
6. 环境装饰和普通 UI。

掉落必须压过普通伤害数字和环境光效。伤害数字应短促、低占屏、快速消散；稀有掉落可使用更强的赤金边光、光柱、图标震动或名称底板。掉落名称和轮廓拥有最高可读权，任何光效不得盖住文字或物品外形。

### 7.4 Inventory & Equipment Layout

背包 / 装备界面应像“铁匠台盘点战利品”，布局稳定、格子清楚、比较直接。

**推荐布局**

- 左侧：角色剪影与装备槽。
- 右侧：背包网格。
- 中区或右侧扩展区：装备详情与新旧装备对比。

背包格、装备槽必须是规则矩形网格，方便鼠标点击、键盘焦点和未来 gamepad 焦点导航。格子边框使用暗金属或旧皮革色，选中、悬停、新物品、可穿戴、可提升状态必须有明确区分。

装备对比必须在 1 秒内回答：是否更强、强在哪里、能不能穿。属性变化不能只靠颜色，必须配合箭头、数值、符号和文字。

```text
攻击 12–18  ▲ +4 提升
防御 1–3    ▼ -1 降低
战力        +18
```

可穿戴装备应使用受控赤金扫边或角标提示；不可穿戴装备使用清晰原因文字，例如“等级不足”“职业不符”。不可穿戴状态不得只用暗色或红色表达。

### 7.5 Typography & Icon Direction

字体要清晰、厚实、有一点碑刻 / 金属感，但数值和说明不能书法化。正文、属性文字、Tooltip、掉落名称优先可读，不为氛围牺牲清晰度。

**文字规则**

- 物品名称、品质、属性、按钮文字必须在暗底上高对比显示。
- 正文优先米灰或暖灰，避免纯白长期刺眼。
- 关键成长数值可用赤金，但只在提升、穿戴成功、奖励确认时使用。
- Tooltip 和掉落名称必须放在低噪声底板上。

**图标规则**

- 装备图标采用半写实剪影 + 手绘材质感 + 清晰轮廓。
- 图标优先表现类别轮廓：剑、斧、甲、盔、戒指、药水等必须在小尺寸下可识别。
- 装备图标不堆复杂细节，避免缩小后变成噪声。
- 品质不只靠颜色，还要靠边框厚度、角标、光效强度、符号区分。

### 7.6 UI Motion & Feedback

UI 动效应短促、厚重、有重量，像金属敲击、火光扫过、余烬消散。动效用于确认操作与强化成长感，不用于制造轻飘装饰。

**推荐动效关键词**

- 短促。
- 厚重。
- 有重量。
- 火光扫过。
- 金属敲击。
- 余烬消散。

**禁止方向**

- 果冻弹跳。
- 彩虹 Hover。
- 科幻电流。
- 玻璃高光。
- 抽卡式全屏转场。

**穿戴反馈推荐流程**

1. 图标从背包飞向装备槽。
2. 装备槽赤金扫边。
3. 属性 / 战力数字弹出。
4. 角色或装备剪影短暂被暖光照亮。
5. 光效收束，回到暗色稳定状态。

拾取反馈应更短：掉落点到角色 / 背包入口出现赤金粒子或短线回流，随后出现简短拾取提示。普通拾取不应达到穿戴成功或稀有爆装的视觉强度。

### 7.7 Accessibility & UX Constraints

UI/HUD 必须保留厚重铁匠台质感，但只要与可读性、点击命中、装备对比冲突，就必须牺牲装饰性。

硬约束：

- 赤金只用于成长、奖励、关键确认。
- 所有关键状态必须有非颜色备份：图标、文字、符号、边框、动画节奏。
- 背包格建议不低于 48×48 px。
- 普通按钮建议不低于约 32 px 高。
- Tooltip、掉落名称、属性数值必须放在低噪声底板上。
- 焦点状态必须可见，预留键盘 / gamepad 导航。
- 装备对比在灰度下仍必须能判断提升 / 降低。
- 如果厚重装饰影响可读性、点击命中、装备比较，删装饰。

### 7.8 Practical Review Tests

1. **30 秒闭环测试**：玩家能否顺畅完成刷怪、掉落、拾取、判断、穿戴、看到变强。
2. **3 秒掉落识别测试**：玩家能否立刻指出最重要掉落。
3. **1 秒背包扫读测试**：玩家能否看出新物品、可穿戴装备、背包状态。
4. **装备对比无颜色测试**：灰度下仍能判断提升 / 降低。
5. **低亮度 HUD 测试**：暗色矿洞中 HUD 和文字仍清楚。
6. **点击命中测试**：按钮、格子、掉落不要求精确像素点击。
7. **赤金稀缺性测试**：普通状态少赤金，掉落和穿戴时赤金明显升权。

## 8. Asset Standards

本节定义《新玛法：觉醒客户端》Phase 1 资产生产、导入、命名、图集、特效和验收标准。Phase 1 的资产目标不是大而全，而是稳定支撑 30 秒离线刷怪爆装切片：**点击移动 → 打怪 → 掉落 → 拾取 → 背包判断 → 穿戴 → 看到变强**。

核心资产规则：**读得清、跑得稳、可快速迭代。** 任何资产规格都必须服务第 1 节视觉身份：暗色玛法世界作为底，赤金成长火光作为奖励焦点；掉落、装备、成长反馈拥有最高视觉优先级。

技术优先级：**掉落识别 > 60 fps 稳定 > Phase 1 内存预算 > draw call 控制 > 美术细节 > 单个资产炫技。** 当美术理想与技术预算冲突时，Phase 1 采用可承受版本；高规格版本可记录为 Phase 2 或 Vertical Slice Plus 增强项。

### 8.1 Scope & Performance Constraints

本标准适用于 Godot 4.6.3 + GDScript 的 2D / 2.5D PC 原生客户端，使用 Godot 2D / CanvasItem 管线。

Phase 1 性能预算：

| 目标 | 标准 |
|---|---:|
| Target Framerate | 60 fps |
| Frame Budget | 16.6 ms |
| Phase 1 Draw Calls | 2D frame draw calls comfortably under 500 |
| Phase 1 Client Memory | Under 1 GB RAM |

资产制作必须避免以下风险：

- 不必要的 4096×4096 超大图集。
- 大量透明空白区域。
- 同屏长期存在的大面积半透明 VFX。
- 每个物品、怪物或 UI 状态使用独立 Shader / Material 实例。
- 环境火光、矿石高光或 UI 装饰抢走掉落识别度。
- 通过 Godot 节点缩放长期修正资产比例。

### 8.2 File Formats

正式进入游戏的 2D 资产以 **PNG** 为主交付格式。

| 资产类型 | 推荐源文件 | 游戏导入文件 | 说明 |
|---|---|---|---|
| 角色 / 怪物 Sprite | `.aseprite` / `.psd` / `.kra` | `.png` / sprite sheet | 源文件保留分层，导出透明 PNG |
| 装备 / 物品图标 | `.psd` / `.aseprite` / `.svg` | `.png` | 游戏内不依赖运行时矢量缩放 |
| UI 面板 / 按钮 | `.psd` / `.png` / `.svg` | `.png` | 九宫格面板需标注拉伸区域 |
| Tile / 环境 | `.aseprite` / `.psd` / `.png` | `.png` / tileset atlas | 统一像素密度和导入设置 |
| VFX 序列帧 | `.aseprite` / `.png` sequence | `.png` sprite sheet | 火光、掉落闪光、命中特效优先序列帧 |
| 参考图 / 概念图 | `.psd` / `.png` | `.png` | 可不直接进入游戏包体 |

规则：

- JPG 不用于角色、UI、图标、掉落物或 VFX。
- 透明 PNG 边缘必须无黑边、白边、脏边。
- 源文件、导出文件和评审预览图应可追溯。
- 正式资产不得只保留扁平 PNG，除非该资产确定不再修改。

### 8.3 Naming & Directory Rules

所有资产文件名使用 `snake_case`。

推荐格式：

```text
[category]_[name]_[variant]_[size]_[index].ext
```

示例：

```text
char_warrior_idle_s_0001.png
char_warrior_attack_e_0003.png
mob_cave_zombie_walk_n_0004.png
item_sword_iron_common_64.png
drop_sword_iron_ground_48.png
vfx_drop_rare_glow_001.png
tile_mine_floor_dark_01.png
ui_inventory_slot_hover_64.png
```

品质文件名可使用英文 slug，但必须映射到第 4.3 的中文品质体系：`common=普通`、`excellent=优秀`、`rare=稀有`、`epic=史诗`、`legend=传说`、`mythic=神话 / 特殊`。不得使用 `magic` 作为正式品质名，以免与“优秀”或“稀有”混淆。

推荐前缀：

| 前缀 | 用途 |
|---|---|
| `char_` | 玩家角色 |
| `mob_` | 怪物 |
| `npc_` | NPC |
| `item_` | 物品 / 装备图标 |
| `drop_` | 地面掉落物 |
| `tile_` | 地图 Tile |
| `env_` | 环境装饰 |

### 8.4 Resolution Tiers

Phase 1 采用清晰、克制、可扩展的分辨率策略。常用显示尺寸必须在源资源中正确制作，不依赖运行时大幅缩放。

| 资产 | 推荐尺寸 / 范围 | 说明 |
|---|---:|---|
| 背包 / 装备图标 | 64×64 | Phase 1 主基准 |
| 重点装备展示 | 96×96 / 128×128 | 装备详情、拾取弹窗、奖励展示 |
| 小型状态图标 | 32×32 | Buff、简易状态、小提示 |
| 地面掉落物 | 32×32 / 48×48 / 64×64 | 本体可小，需配合光圈或名称识别 |
| 玩家战士单帧 | 96×96–160×160 | 预留武器挥砍空间，脚底锚点稳定 |
| 普通矿洞怪单帧 | 80×80–128×128 | 剪影低于玩家复杂度，不抢掉落 |
| 精英 / 高威胁怪 | 128×128–192×192 | Phase 1 可选 |
| 矿洞 Tile | 64×64 | 推荐主基准，适合 PC 观感和 TileMap 组织 |
| 小型 VFX | 64×64–128×128 | 命中、拾取、普通掉落 |
| 重点 VFX | 128×128–192×192 | 稀有掉落、穿戴成功、强化成功 |

可读性检查必须覆盖：

1. 100% 显示：正式标准。
2. 75% 显示：仍能识别类别和品质。
3. 125% 显示：不显脏、不糊、不暴露边缘瑕疵。

### 8.5 Atlas / Tileset / Sprite Sheet Standards

图集目标是减少碎片、稳定导入、降低 draw call 风险，但不同用途、不同过滤设置、不同 Shader 需求的资产不得无脑混进同一张图。

推荐拆分：

| Atlas 类型 | 内容 | 推荐尺寸 |
|---|---|---:|
| UI Core Atlas | 按钮、边框、面板角、通用 UI 图标 | 1024×1024 |
| Item Atlas | 装备、材料、金币、药水 | 1024×1024 |
| Environment Atlas | 矿洞地表、墙体、矿石、碎石 | 1024×1024 / 2048×2048 |
| Character Atlas | 战士动作序列 | 2048×2048 |
| Monster Atlas | 矿洞怪物动作序列 | 2048×2048 |
| VFX Atlas | 火光、掉落闪光、命中反馈 | 1024×1024 / 2048×2048 |

硬规则：

- UI、角色、怪物、环境、物品、VFX 分 atlas 管理。
- Phase 1 优先 1024 / 2048 图集。
- 4096×4096 只在必要且验证后使用；单张 4096 RGBA8 纹理约 64 MB，不能随意引入。
- 不同过滤设置的资产不要混图集。
- 普通 Sprite 与特殊发光材质尽量拆分，避免批处理失效。
- 图集帧之间保留至少 2px padding；线性过滤资源需考虑 extrude 边缘。

角色 / 怪物动画建议按动作拆分，便于 Phase 1 快速调整：

```text
char_warrior_idle_sheet_medium.png
char_warrior_walk_sheet_medium.png
char_warrior_attack_sheet_medium.png
char_warrior_hit_sheet_medium.png
mob_cave_zombie_idle_sheet_medium.png
mob_cave_zombie_walk_sheet_medium.png
mob_cave_zombie_attack_sheet_medium.png
mob_cave_zombie_death_sheet_medium.png
```

每个 sprite sheet 应记录：帧宽、帧高、帧数、FPS、Pivot、方向需求和是否包含 padding。

Phase 1 动画帧数建议：

| 动作 | 帧数建议 |
|---|---:|
| Idle | 4–8 |
| Walk / Run | 6–8 |
| Attack | 6–10 |
| Hit / Hurt | 2–4 |
| Death | 6–10 |

普通怪物总帧数不建议超过 64；Phase 1 玩家战士总帧数不建议超过 128。

### 8.6 Godot Import Standards

Godot 4.6.3 是项目固定版本，Importer 选项必须以项目实际 Import 面板为准；若 UI 名称变化，以本节目标为准。

普通 2D Sprite：

- Filter：默认关闭，除非该资源需要平滑缩放。
- Mipmaps：默认关闭。
- Repeat：默认关闭。
- Compression：根据透明度和清晰度测试决定，禁止无理由保留超大未压缩纹理。

UI 图标 / UI 纹理：

- 图标必须在目标显示尺寸下清晰。
- 同一 UI 模块不得混用锐利和模糊的导入策略。
- 9-slice 框体需要保留足够边缘区域，避免拉伸破坏铁匠台质感。

VFX / 光效纹理：

- Filter 通常可开启，以保证柔光、烟雾、火焰平滑。
- Mipmaps 默认关闭，只有出现明显缩放闪烁时再评估。
- 透明区域必须裁切，禁止整张大透明贴图。
- 光斑、火舌、冲击波优先进入共享 VFX 图集。

导入后必须检查：

- 动画帧是否错位。
- UI 图标是否串图。
- 掉落物是否显示正确。
- 材质和 Shader 是否仍绑定正确。
- Y-sort 和遮挡是否未被破坏。

### 8.7 Material / Shader Budget

普通 Sprite 默认不使用自定义 Shader。Phase 1 自定义 Shader 仅允许用于：

- 掉落品质光效。
- 赤金成长 / 强化反馈。
- 简单受击闪白。
- 怪物死亡淡出或简单溶解。
- UI 高亮、描边、扫光。

限制：

- 禁止为每个物品、每个怪物创建独立 Shader 变体。
- 优先共享 Shader + 参数控制。
- Shader 参数变化过多可能影响批处理，应减少独立 Material 实例。
- Phase 1 默认不使用高成本全屏后处理。
- 禁止在普通背景物件上使用发光 / 动态材质抢夺掉落焦点。

### 8.8 VFX / Overdraw Budget

VFX 必须短促、意图直接、层级明确。成长类 VFX 比战斗类 VFX 更暖、更亮；战斗类 VFX 不得遮挡掉落判断。

| VFX 类型 | 同屏建议上限 | 单个效果粒子上限 | 说明 |
|---|---:|---:|---|
| 普通打击火花 | 10 个效果 | 8–16 | 短生命周期 |
| 怪物死亡效果 | 5 个效果 | 16–32 | 死亡后让位于掉落 |
| 普通掉落出现 | 10 个效果 | 8–16 | 重点是识别，不是遮挡 |
| 稀有掉落光效 | 3–5 个效果 | 16–32 | 可更醒目，但需限制数量 |
| 环境氛围粒子 | 1–2 个系统 | 32–64 | 低透明度、低频率 |

Overdraw 规则：

- 烟雾、光圈、火焰、冲击波不得长期覆盖大面积屏幕。
- 同一位置不得叠加超过 3 层高透明度 VFX。
- 赤金成长特效集中在角色、掉落和 UI 反馈，不铺满地图。
- 全屏泛光、屏幕暗角、色彩覆盖等后处理 Phase 1 默认禁止。

### 8.9 Drop FX Standards

掉落光效必须服务“爆装有戏”，但不能遮挡物品本体、掉落名称或价值判断。

| 品质 / 重要度 | 允许效果 | 技术限制 |
|---|---|---|
| 普通掉落 | 静态小图 + 极弱阴影 / 轮廓 | 无动态光效或极低频闪 |
| 较好掉落 | 小型呼吸光圈 | 单层、低透明度 |
| 稀有掉落 | 品质色光圈 + 短粒子 | 限制同屏数量 |
| 核心奖励 | 赤金爆点 + 持续轻量光效 | 只允许少量同时存在 |

硬限制：

- 同屏持续发光掉落物建议不超过 20 个。
- 同屏稀有掉落强光效建议不超过 5 个。
- 单个掉落持续光效最多包含 1 个底部光圈、1 个轻量呼吸层和少量粒子。
- 掉落出现峰值应在 0.3–0.8 秒内衰减。
- 怪物死亡同时爆多个物品时，优先保证分布、图标和名称识别，不允许用大面积光爆盖住地面。

### 8.10 Y-sort / Occlusion Requirements

所有角色、怪物、掉落物和可遮挡场景物必须定义统一视觉锚点。

| 资产类型 | 锚点 |
|---|---|
| 角色 / 怪物 | 脚底中心 |
| 掉落物 | 落地点中心 |
| 树 / 柱 / 门框 / 墙角 | 标明可行走底部、遮挡范围和 Y-sort 参考点 |

可遮挡资产建议拆为三层：

1. 底层：地面接触、阴影、不可遮挡部分。
2. 实体层：参与 Y-sort 的主体部分。
3. 前景层：始终压在角色上方的遮挡部分，例如树冠、屋檐、洞口前沿。

规则：

- 禁止将大面积建筑做成单张不可拆分 Sprite，导致 Y-sort 错误。
- 遮挡物不得完全隐藏掉落物关键识别信息。
- 掉落物位于遮挡区域时，必须保留轮廓、光圈、名称或拾取提示。
- 暗色前景遮挡不得与角色轮廓完全融合。

### 8.11 Phase 1 Minimum Asset Pack

Phase 1 最小资产包用于验证 30 秒刷怪爆装技术切片。

角色：

- 1 套战士基础动作 sprite sheet。
- 1 套基础武器表现。
- 1 套角色阴影。

怪物：

- 1 种矿洞普通怪完整动作。

- 8–12 个矿洞装饰物。
- 2–3 个火光 / 余烬氛围物件。

掉落与装备：

- 8–12 个装备 / 道具图标。
- 普通 / 优秀 / 稀有三档品质框。
- 金币或基础货币图标。
- 2–3 种地面掉落显示。
- 普通掉落与稀有掉落光效。

VFX：

- 普通命中。
- 强命中 / 暴击。
- 怪物死亡。

### 8.12 Asset Review Tests

每批正式资产进入项目后，必须通过以下检查：

1. **黑底测试**：在暗背景下仍能识别角色、怪物、掉落和 UI 图标。
2. **矿洞实景测试**：放入 Phase 1 矿洞场景后，掉落和角色不被环境吞掉。
3. **缩放测试**：75% / 100% / 125% 显示下均保持可读。
4. **掉落识别测试**：截图缩小到 50% 后，玩家仍能第一眼看出哪里有重要掉落。
5. **背包扫读测试**：玩家 1 秒内能看出新物品、品质、可穿戴状态和提升方向。
6. **灰度测试**：去掉颜色后，装备品质、危险、可交互和成长完成仍有形状 / 图标 / 文字备份。
7. **Draw Call 检查**：场景 draw calls 仍舒适低于 500。
8. **Overdraw 检查**：掉落、命中、死亡和环境火光同时出现时没有大面积半透明堆叠。
9. **Import 检查**：图集、SpriteFrames、AtlasTexture、Material 和 Shader 引用稳定。
10. **Y-sort 检查**：角色、怪物、掉落物在矿车、木支架、岩壁附近不被错误遮挡。

资产验收标准不是“单张图是否精美”，而是它是否让玩家在 30 秒内明确感到：**我在稳刷，我看见爆装，我知道什么更好，我愿意再刷一轮。**

## 9. Reference Direction

本节定义《新玛法：觉醒客户端》的视觉参考方向。参考源用于约束美术判断，不用于直接复制画面、资产、UI 或 IP 元素。所有参考都必须回到本项目核心视觉规则：**暗色玛法世界为底，赤金成长火光为奖励焦点；玩家必须第一眼看见掉落、理解价值、确认变强。**

### 9.1 经典传奇 / OpenMir2

**Draw From**

- 职业、刷怪、爆装、地图推进和装备崇拜的经典骨架。
- 暗色地图、粗粝矿洞、低饱和世界和装备掉落的期待感。
- 战士、法师、道士的基础视觉身份：武器、长袍、符纸 / 召唤辅助轮廓。
- 城镇作为安全据点、矿洞作为刷怪容器、装备作为长期追求目标的视觉逻辑。

**Avoid**

- 不直接复刻原始资源、地图、图标或 UI。
- 不保留低清、脏乱、难读、信息层级不清的问题。
- 不把怀旧等同于落后；经典骨架必须被现代清晰度重绘。
- 不让 OpenMir2 之外的临时客户端或复刻项目成为行为 / 视觉权威。

**Project Rule**

如果“复古质感”和“掉落 / UI 可读性”冲突，保留传奇骨架，但用现代可读性重绘。

### 9.2 暗黑刷装 ARPG 的战利品可见性

**Draw From**

- 掉落在暗色环境中拥有明确视觉优先级。
- 品质、价值和拾取欲望通过颜色、光效、音效、名称和节奏共同表达。
- 战斗空间中怪物、地面、特效必须为掉落识别让路。
- 高价值掉落需要短促但明确的情绪峰值。

**Avoid**

- 不追求满屏爆炸、复杂词缀墙或过度粒子噪声。
- 不让掉落特效遮挡物品名称、地面位置或实际价值判断。
- 不采用过暗、过血腥、过写实的哥特恐怖方向。
- 不把刷装爽感变成纯视觉轰炸。

**Project Rule**

掉落要“有戏”，但必须先可读；稀有掉落可以夸张，普通掉落必须克制。

### 9.3 原神 / 崩坏：星穹铁道的现代成长反馈

**Draw From**

- 清晰的角色成长、装备提升、阶段目标和奖励确认反馈。
- UI 信息层级：玩家能快速看懂“获得了什么、强在哪里、下一步做什么”。
- 养成结果的仪式感：获得、装备、提升、完成目标时有明确音画确认。
- 长期目标被拆成清晰可追踪的小目标。

**Avoid**

- 不采用开放世界探索优先。
- 不采用抽卡作为核心奖励来源。
- 不使用过度明亮、干净、二次元、科幻或玻璃感 UI。
- 不让菜单养成压过“打怪爆装”的核心体验。

**Project Rule**

可以学习现代成长反馈的清晰度，但奖励来源必须回到刷怪、掉落、穿戴和角色变强。

### 9.4 阴阳师的长期刷取与价值判断

**Draw From**

- 长期刷取、筛选、比较、保留、替换的养成节奏。
- 物品价值判断需要清楚的品质、属性、可用性和提升提示。
- 玩家应能快速分辨“值得留”“可以换上”“可以卖掉 / 分解”的物品。
- 重复刷取也需要阶段性目标和小型成就反馈。

**Avoid**

- 不采用纯抽卡收集作为主循环。
- 不做复杂到 Phase 1 无法承载的多词条深养成。
- 不让背包整理变成刷怪流的长期阻塞点。
- 不让装备判断只靠颜色或稀有度，而忽视实际属性提升。

**Project Rule**

背包和装备界面必须像铁匠台盘点战利品：清楚、扎实、可比较，而不是华丽但慢。

### 9.5 铁匠铺 / 火光 / 黑铁器物

**Draw From**

- 火光照亮铁器的局部高光：热、亮、短促、扎实。
- 黑铁、旧皮革、石板、铆钉、磨损边、余烬、火星。
- 装备从旧铁到赤金高光的成长路径。
- 铁匠台作为 UI 隐喻：装备被放在暗色台面上检视，属性提升像火光扫过。

**Avoid**

- 不做明亮童话式锻造间。
- 不让火光铺满整个世界，削弱成长反馈的稀缺性。
- 不把铁器质感做成脏到看不清的黑灰噪声。
- 不使用现代科幻金属、霓虹灯、玻璃拟态或塑料质感。

**Project Rule**

赤金火光只属于成长、奖励、强化、穿戴成功和关键确认；普通环境火光必须更暗、更慢、更克制。

### 9.6 矿洞 / 旧石城 / 战后玛法

**Draw From**

- 矿洞作为 Phase 1 刷怪容器：黑铁、岩灰、煤尘、断梁、铁轨、矿车、火盆、塌方。
- 旧石城作为安全据点：厚墙、旧旗、炉火、商人、仓库、铁匠铺。
- 世界有历史伤痕，但叙事细节必须服从刷怪路线和掉落识别。
- 暗色环境通过冷暖对比让奖励光更有价值。

**Avoid**

- 不做复杂迷宫、强探索、密集遮挡。
- 不使用大面积发光矿石、宝石堆或亮色装饰，以免混淆掉落。
- 不通过大量碑文、剧情物件或机关暗示增加 Phase 1 复杂度。
- 不让环境叙事压过移动、打怪、掉落和拾取。

**Project Rule**

Phase 1 矿洞首先是刷怪爆装容器，其次才是叙事场景。

### 9.7 Reference Priority Order

当不同参考方向冲突时，按以下顺序决策：

1. **OpenMir2 / 经典传奇骨架**：决定职业、刷怪、爆装、地图推进和玛法识别感。
2. **本项目 Art Bible 第 1 节视觉规则**：决定暗色世界和赤金成长火光的主视觉。
3. **现代成长反馈参考**：决定 UI 清晰度、奖励确认和目标引导。
4. **铁匠铺 / 黑铁 / 火光参考**：决定材质、UI 隐喻和装备成长质感。
5. **其他刷装 ARPG 参考**：只借鉴掉落可见性和奖励节奏，不覆盖传奇骨架。

### 9.8 Final Reference Test

任何新增参考图、游戏、电影或美术方向，都必须通过以下问题：

1. 它是否让玩家更快看见掉落？
2. 它是否让装备价值更容易判断？
3. 它是否让成长反馈更热、更亮、更直接、更可读？
4. 它是否保留传奇骨架，而不是把项目推向其他类型？
5. 它是否能在 Godot 2D / CanvasItem Phase 1 预算内实现？
6. 它是否避免侵犯原始资源、地图、图标、角色或 UI 的版权 / IP 风险？

如果答案不能服务“稳刷、爆装、穿戴、变强”，该参考只能作为氛围参考，不能成为生产标准。
